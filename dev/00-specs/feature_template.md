## 🔧 使用说明

### 创建新 Feature 的步骤

1. 复制上述提示词模板
2. 根据具体需求填写模板中的占位符
3. 确保用户故事清晰且有价值
4. 验收标准要具体且可验证
5. 在对应 Epic 目录下创建 Feature 目录
6. 将生成的内容保存为 `README.md`

### 质量检查清单

- [ ] 功能模块定义清晰且完整
- [ ] 用户场景真实且有价值
- [ ] 验收标准具体且可测试
- [ ] Task 分解合理且可执行
- [ ] 依赖关系明确且准确
- [ ] 成功指标可衡量且有意义

### 注意事项

- Feature 应该是独立的功能模块，可以单独交付价值
- 用户故事要从用户角度出发，体现真实需求
- 验收标准要覆盖功能、性能、安全等多个维度
- Task 分解要考虑开发团队的实际能力
- 定期更新进度和风险状态
